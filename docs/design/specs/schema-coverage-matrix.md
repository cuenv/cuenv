# Schema Coverage Matrix

This matrix is the agent-facing source of truth for cuenv feature coverage. It is intentionally schema-first: every exported `#Definition` from `schema/**/*.cue` has a row.

Status values:

- `implemented`: schema, Rust implementation, docs, and examples are aligned enough for agents to recommend the feature.
- `partial`: schema exists and some implementation exists, but agents must call out missing fields, provider gaps, or command limitations.
- `schema-only`: schema exists but the current implementation should not be presented as usable.
- `legacy`: retained for compatibility or older docs, not the preferred surface.
- `internal`: helper type that agents should usually explain only when debugging the schema or implementation.
- `docs-misleading`: current docs or examples overclaim support and must be corrected before agents rely on them.
- `needs-decision`: ownership or intended user surface is unclear.
